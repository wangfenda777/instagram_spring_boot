## 二、命名规范

### 2.1 包命名
- 全部小写
- 多个单词用点分隔
- 示例：`com.example.instagram.module.user.service`

### 2.2 类命名

| 类型 | 命名规则 | 示例 |
|-----|---------|------|
| Controller | `{模块名}Controller` | `UserController` |
| Service 接口 | `{模块名}Service` | `UserService` |
| Service 实现 | `{模块名}ServiceImpl` | `UserServiceImpl` |
| Mapper | `{实体名}Mapper` | `UserMapper` |
| Entity | `{实体名}` | `User`, `Post` |
| DTO | `{功能}DTO` | `LoginDTO`, `UserRegisterDTO` |
| VO | `{功能}VO` | `UserInfoVO`, `PostDetailVO` |
| 常量类 | `{模块}Constant` | `AuthConstant`, `RedisKeyConstant` |
| 枚举类 | `{含义}Enum` | `MediaTypeEnum`, `NotificationTypeEnum` |
| 工具类 | `{功能}Util` | `JwtUtil`, `PasswordUtil` |
| 异常类 | `{含义}Exception` | `BusinessException`, `AuthException` |

### 2.3 方法命名

| 操作类型 | 命名规则 | 示例 |
|---------|---------|------|
| 查询单个 | `get{实体}By{条件}` | `getUserById`, `getUserByUsername` |
| 查询列表 | `list{实体}By{条件}` | `listPostsByUserId` |
| 分页查询 | `page{实体}By{条件}` | `pagePostFeed` |
| 新增 | `create{实体}` | `createPost`, `createUser` |
| 更新 | `update{实体}` | `updateUserProfile` |
| 删除 | `delete{实体}By{条件}` | `deletePostById` |
| 统计 | `count{实体}By{条件}` | `countFollowersByUserId` |
| 判断 | `is{条件}` / `has{条件}` | `isFollowing`, `hasUnreadNotification` |

### 2.4 变量命名
- 驼峰命名法
- 布尔类型以 `is` / `has` 开头
- 集合类型以复数形式命名
- 示例：`userId`, `isVerified`, `postList`

## 三、代码分层规范

### 3.1 Controller 层
- 负责接收请求、参数校验、调用 Service、返回响应
- 不包含业务逻辑
- 使用 `@RestController` 注解
- 统一返回 `Result<T>` 格式

```java
@RestController
@RequestMapping("/api/user")
public class UserController {
    
    @Autowired
    private UserService userService;
    
    @GetMapping("/me")
    public Result<UserInfoVO> getCurrentUser() {
        return Result.success(userService.getCurrentUser());
    }
}
```

### 3.2 Service 层
- 负责业务逻辑处理
- 接口定义在 `service` 包，实现在 `service.impl` 包
- 事务控制在 Service 层

```java
public interface UserService {
    UserInfoVO getCurrentUser();
}

@Service
public class UserServiceImpl implements UserService {
    
    @Autowired
    private UserMapper userMapper;
    
    @Override
    public UserInfoVO getCurrentUser() {
        // 业务逻辑
    }
}
```

### 3.3 Mapper 层
- 负责数据库操作
- 继承 `BaseMapper<T>`
- 复杂查询使用 `@Select` 注解或 XML 映射

```java
@Mapper
public interface UserMapper extends BaseMapper<User> {
    
    @Select("SELECT * FROM users WHERE username = #{username}")
    User selectByUsername(String username);
}
```

### 3.4 Entity 层
- 对应数据库表结构
- 使用 `@TableName` 指定表名
- 使用 `@TableId` 指定主键
- 使用 Lombok 简化代码

```java
@Data
@TableName("users")
public class User {
    
    @TableId(type = IdType.AUTO)
    private Long id;
    
    private String username;
    private String email;
    
    @TableField("created_at")
    private LocalDateTime createdAt;
}
```

### 3.5 DTO 层（Data Transfer Object）
- 用于接收前端请求参数
- 包含参数校验注解

```java
@Data
public class LoginDTO {
    
    @NotBlank(message = "账号不能为空")
    private String account;
    
    @NotBlank(message = "密码不能为空")
    private String password;
}
```

### 3.6 VO 层（View Object）
- 用于返回给前端的数据
- 只包含前端需要的字段
- ID 类型使用 String（避免 JS 精度丢失）

```java
@Data
public class UserInfoVO {
    private String userId;
    private String username;
    private String displayName;
    private String avatar;
}
```

## 四、统一响应格式

### 4.1 Result 类

```java
@Data
public class Result<T> {
    private Integer code;
    private String message;
    private T data;
    private Long timestamp;
    
    public static <T> Result<T> success(T data) {
        Result<T> result = new Result<>();
        result.setCode(200);
        result.setMessage("success");
        result.setData(data);
        result.setTimestamp(System.currentTimeMillis());
        return result;
    }
    
    public static <T> Result<T> error(Integer code, String message) {
        Result<T> result = new Result<>();
        result.setCode(code);
        result.setMessage(message);
        result.setTimestamp(System.currentTimeMillis());
        return result;
    }
}
```

### 4.2 分页响应

```java
@Data
public class PageResult<T> {
    private List<T> list;
    private Long total;
    private Integer page;
    private Integer pageSize;
    private Boolean hasMore;
}
```

## 五、异常处理规范

### 5.1 自定义异常

```java
public class BusinessException extends RuntimeException {
    private Integer code;
    
    public BusinessException(Integer code, String message) {
        super(message);
        this.code = code;
    }
}
```

### 5.2 全局异常处理

```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    
    @ExceptionHandler(BusinessException.class)
    public Result<?> handleBusinessException(BusinessException e) {
        return Result.error(e.getCode(), e.getMessage());
    }
}
```

## 六、数据库操作规范

### 6.1 ID 类型处理
- 数据库使用 `BIGINT`
- Entity 使用 `Long`
- VO/DTO 使用 `String`（通过 `@JsonSerialize` 转换）

### 6.2 时间字段处理
- 数据库使用 `TIMESTAMP`
- Entity 使用 `LocalDateTime`
- VO 返回时间戳（Long 类型）

### 6.3 逻辑删除
- 使用 `is_deleted` 字段
- MyBatis-Plus 配置逻辑删除

### 6.4 字段映射
- 数据库字段使用下划线命名：`created_at`
- Java 字段使用驼峰命名：`createdAt`
- MyBatis-Plus 自动映射

## 七、接口开发流程

### 7.1 开发步骤
1. 阅读接口文档（`doc/` 目录）
2. 创建/确认 Entity（对应数据库表）
3. 创建 Mapper 接口
4. 创建 DTO（请求参数）和 VO（响应数据）
5. 创建 Service 接口和实现类
6. 创建 Controller
7. 添加 Knife4j 接口文档注解
8. 测试接口

### 7.2 模块开发顺序
建议按以下顺序开发模块：
1. 公共模块（common）
2. 认证模块（auth）
3. 用户模块（user）
4. 帖子模块（post）
5. 关注模块（follow）
6. 快拍模块（story）
7. 通知模块（notification）
8. 探索模块（explore）
